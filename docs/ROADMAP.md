# Cover Maker 长期路线图（2026-2028）

## 1. 项目愿景
Cover Maker 的目标不是“能出图”的工具，而是成为一款：
- 本地优先、离线可用、低学习门槛的封面设计工作台
- 面向内容创作者和运营同学的“高效率生产工具”
- 可持续演进到桌面端、插件生态、批量自动化与协作能力的平台

## 2. 当前基线（2026-03）
当前版本已具备可交付能力：
- 画布编辑：文本、图片、形状、图层顺序
- 模板系统：内置模板 + JSON 导入导出
- 交互增强：吸附对齐线、图层管理、独立缩放、中键平移
- 字体系统：分类、搜索、收藏、最近使用，多字体离线打包
- 导出与保存：PNG 导出、撤销重做、本地草稿持久化
- 适配与性能：1K/2K/4K 屏幕适配，基础渲染与快照性能优化

## 3. 北极星指标（North Star）
后续迭代围绕以下指标衡量：
- 首图完成时间（新用户）：`<= 3 分钟`
- 连续编辑流畅度：`60FPS` 为目标，复杂场景最低不低于 `30FPS`
- 导出成功率：`>= 99.5%`
- 崩溃率（会话）：`< 0.5%`
- 模板复用率：`>= 40%`（模板被二次修改并导出的比例）

## 4. 里程碑规划

### M4（2026 Q2）稳定性与发布就绪
目标：从“可用”走向“稳定可长期使用”。

功能与工程项：
- 网格/安全边距参考线
- 文本增强：行高、字间距、描边、阴影
- 图片增强：裁切、圆角、透明度、滤镜基础
- 历史栈优化：大文档下撤销重做内存控制
- 自动保存策略升级：版本化草稿与损坏恢复

工程治理：
- 单元测试（关键工具函数）
- 编辑器关键流程回归测试（Playwright）
- 发布前 checklist（字体、导出、模板兼容、快捷键）

验收标准：
- 50+ 图层场景可编辑
- 核心功能回归自动化覆盖到位
- README 与文档可以支持新用户独立上手

### M5（2026 Q3-Q4）专业化编辑能力
目标：提升模板产出质量与高级编辑能力。

功能方向：
- 智能对齐增强：等距分布、智能吸附提示
- 形状系统扩展：线条、箭头、多边形、布尔操作（基础）
- 组件能力：元素分组/解组、锁定组、复制样式
- 模板版本：模板元数据、标签、封面预览图
- 导出增强：JPEG/WebP、质量参数、批量导出

性能方向：
- 大对象列表虚拟化
- 主线程重任务下沉（Web Worker）
- Fabric 模块动态分包，降低首屏体积

验收标准：
- 模板设计效率明显提升（内部体验评估）
- 首次加载时间和交互延迟下降

### M6（2027 H1）桌面化与资源生态
目标：打造“真正可双击使用”的本地设计工具。

功能方向：
- Tauri 桌面端打包（Windows/macOS/Linux）
- 本地素材库：背景、贴纸、图标分类
- 字体包管理：本地导入、启用/禁用、许可证提醒
- 工程文件格式：`cover-maker-project-v1`（可包含资源索引）

工程方向：
- 跨平台文件路径与权限统一适配
- 工程文件向后兼容策略
- 错误日志与崩溃恢复机制

验收标准：
- 桌面端安装包可稳定运行
- 项目文件跨设备可迁移并保持兼容

### M7（2027 H2）协作与自动化生产
目标：从单人工具升级为团队生产工具。

功能方向：
- 批量生成封面（CSV/JSON 变量驱动）
- 团队模板规范（字段锁定、品牌色、字体策略）
- 操作录制与宏（重复动作自动化）
- 多模板流水线导出（适配不同平台尺寸）

工程方向：
- 任务队列和并发导出调度
- 配置模板 schema 与校验器
- 失败重试与结果报告

验收标准：
- 一次配置可批量生成多张封面
- 团队成员可复用同一规范稳定出图

### M8（2028）平台化与智能化能力
目标：成为“创作系统”而非单一编辑器。

探索方向：
- 插件系统：第三方素材源、导出器、特效插件
- 智能辅助：标题排版建议、配色建议、模板推荐
- 资产市场：模板与素材包分发机制
- 可观测性体系：匿名性能上报（可选、可关闭）

## 5. 技术演进路线

### 5.1 架构分层
- `Editor Core`：画布实体与命令执行
- `State Layer`：业务状态与持久化
- `UI Layer`：面板、工具栏、反馈
- `I/O Layer`：模板/项目导入导出、资源管理

### 5.2 状态与历史
- 引入命令模式（Command Pattern）统一撤销重做
- 历史快照由全量转为“差异 + 关键帧”混合策略

### 5.3 渲染与性能
- 画布操作热点分析（move/scale/text input/export）
- 高成本计算拆分为 worker 任务
- 初始化按需加载（代码分割 + 面板懒加载）

### 5.4 兼容性策略
- 文件格式版本号与迁移器
- 模板兼容矩阵（旧版模板向新版兼容）
- 字体缺失时回退策略

## 6. 质量与交付策略
- 版本节奏：`双周小版本 + 季度稳定版本`
- 质量门禁：`lint + typecheck + build + e2e smoke`
- 文档门禁：功能上线必须同步 README/ROADMAP/CHANGELOG
- 发布流程：预发布验证 -> 版本标签 -> 发布说明

## 7. 风险清单与应对
- 字体版权风险：严格保留来源与许可证文档
- 性能风险：超大画布与多图层导致卡顿，提前设置资源预算
- 兼容风险：模板格式变更导致旧版本不可读，必须有迁移器
- 维护风险：功能膨胀导致复杂度上升，持续模块化拆分

## 8. 未来 8 周执行清单（建议）
- 第 1-2 周：完善 README 教程、错误提示、快捷键面板
- 第 3-4 周：文本高级样式（描边/阴影/字间距）
- 第 5-6 周：图片裁切与滤镜基础
- 第 7-8 周：回归测试与发布流程脚本化

## 9. 完成定义（Definition of Done）
每个功能在合入前必须满足：
- 功能可用：核心流程跑通
- 可回归：至少有最小验证用例
- 可文档化：README/ROADMAP 已同步
- 可恢复：异常情况下不破坏用户草稿
